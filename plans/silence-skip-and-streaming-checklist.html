Key Finding — FluidAudio Streaming API IMPORTANT
FluidAudio ships a complete streaming ASR architecture as part of the SDK already included in TextEcho. StreamingEouAsrManager provides per-chunk partial callbacks via setPartialTranscriptCallback(_:). The streaming model (parakeet-realtime-eou-120m-v1) is a separate download from the TDT V2/V3 batch models — users must opt in. Chunk latency: 160ms (lowest), 320ms, 1280ms. WhisperKit has AudioStreamTranscriber but it owns its own audio tap and conflicts with TextEcho's AudioRecorder — skip for Phase 1.
What Exists Now
Both ParakeetTranscriber.swift and WhisperKitTranscriber.swift apply an RMS gate before calling inference. If audio RMS < 0.005 (≈ −46 dBFS), transcription is skipped entirely and an empty string is returned silently. This was added as a cheap hallucination filter — both models can emit fabricated text on silence. However, the check also silently discards quiet speech (whispered, far-field, soft-spoken).
Why Safe to Remove
WhisperKit already has isHallucination() post-inference
Checks known hallucination phrases ("thank you for watching", etc.) + repetition detection. Covers the silence hallucination case adequately.
Parakeet TDT emits empty text on silence — not hallucinations
The TDT architecture produces no tokens on silent input in practice. The comment "less prone but still possible" in the code was precautionary.
AppState already handles empty transcription cleanly
if text.isEmpty { overlay.hide() } — an empty result from the engine produces the same UX as the RMS skip, but without silently failing on quiet speech.
AudioRecorder's silence detection is separate and untouched
checkSilence() / silenceThreshold in AudioRecorder handles auto-stop. These must NOT be changed.

PHASE A Infrastructure — no visible feature yet Complexity: Low–Medium
Add StreamingTranscriber protocol to Transcriber.swift
New protocol: beginStreaming(), appendAudio(_:sampleRate:), endStreaming() → String, setPartialCallback(_:), supportsStreaming: Bool. Keep existing Transcriber protocol unchanged.
Add onAudioChunk: ((Data) → Void)? to AudioRecorder
Fire in the AVAudioEngine tap callback alongside the existing bufferData.append(). Accumulate tap buffers (~64ms each) into 160ms chunks before firing. The main bufferData accumulation is untouched.
Add .streamingPartial(text: String) to OverlayState
In Overlay.swift. Add showStreamingPartial(_:) to OverlayViewModel and OverlayWindowController.
Add ghost text display to OverlayView for .streamingPartial
Show waveform + growing text below it. Text at ~50% opacity to signal "not yet confirmed". Different colour from final result text.
Add streamingEnabled: Bool to AppConfig
Default false — existing users unaffected. Only Parakeet engine when streaming model is downloaded.
PHASE B ParakeetTranscriber — streaming engine Complexity: High
Add StreamingTranscriber conformance to ParakeetTranscriber
Add streamingManager: StreamingEouAsrManager? property. supportsStreaming returns true.
Implement beginStreaming() — load EOU model if needed
Call StreamingAsrEngineFactory.create(.parakeetEou160ms), then loadModels(). Reset engine state. EOU model stored in its own cache dir (FluidAudio/Models/parakeet-eou-streaming/) — separate from TDT.
Implement appendAudio(_:sampleRate:) — forward chunks to EOU engine
Convert Data (Int16 PCM) to AVAudioPCMBuffer, call streamingManager.appendAudio(_:) then processBufferedAudio() asynchronously. Mind actor isolation — ParakeetTranscriber is an actor, StreamingEouAsrManager is also an actor.
Implement setPartialCallback(_:) — wire to EOU partial callback
Store callback locally; forward to streamingManager.setPartialTranscriptCallback(_:) after init.
Implement endStreaming() → String — flush and return final transcript
Call streamingManager.finish(). Apply existing hallucination text filtering if needed. Reset EOU engine state for next recording.
Add EOU model download status to SettingsWindow
New section under Parakeet engine settings: "Streaming Model" with download button, progress indicator, and cache status. Gate the streamingEnabled toggle on model being downloaded.
Benchmark EOU 120M accuracy vs TDT V3 batch
Test with a range of speech samples. EOU is a different model — verify WER is acceptable before shipping. TDT V3 target is ~2.1% WER.
PHASE C AppState — wiring the streaming pipeline Complexity: Medium
Detect streaming capability in AppState.beginRecording()
Cast transcriber to any StreamingTranscriber and check supportsStreaming and config.model.streamingEnabled.
Wire recorder.onAudioChunk → transcriber.appendAudio() during streaming sessions
Only in streaming mode. In non-streaming mode onAudioChunk remains nil. Clear the wiring in endRecording.
Wire partial callback → overlay.showStreamingPartial(_:)
Ensure @MainActor dispatch when updating overlay from the partial callback (which fires from actor context).
Update endRecording() to call endStreaming() instead of batch transcribe
In streaming mode: let text = try await transcriber.endStreaming(). Use the returned final text for injection, same as batch path. Fallback to batch if streaming returns empty and batch is available.
Handle ESC cancel during streaming session
cancelRecording() must call transcriber.reset() on the streaming engine to clear accumulated state.
PHASE D Overlay UI polish Complexity: Low
Ghost text styling — lower opacity, distinct colour from confirmed result
Suggested: theme.recording.opacity(0.6) for partial text. Final confirmed result uses theme.success as today.
Smooth text update animation — avoid jarring redraws as words appear
Use .animation(.easeIn(duration: 0.1), value: viewModel.resultText) or similar. Test with fast speech.
Show waveform + ghost text simultaneously during streaming
Waveform remains visible (recording is ongoing). Ghost text appears below it. Ensure overlay does not grow too tall for short sentences.
Update status text to "STREAMING" during streaming recording
Distinguish from "RECORDING" (batch mode) so users know what mode they are in.
Feature 1 — Silence Skip Risks
Risk Severity Mitigation
WhisperKit hallucinates on near-silent audio Medium isHallucination() post-inference filter already covers this. Known phrases and repetition loops are caught.
Parakeet returns spurious tokens on silence Low TDT architecture emits empty text on silence. Empty string → overlay hides. No user-visible impact.
Wasted Neural Engine inference on empty recordings Low Inference is 50–200ms. Cost of accidental silent recording is small and rare.
LLM mode receives hallucinated text on silent recording Medium AppState already checks text.isEmpty before entering LLM path. WhisperKit's hallucination filter runs before LLM.
Feature 2 — Streaming Risks
Risk Severity Mitigation
EOU model download burden on users Medium Opt-in via streamingEnabled flag. Clear UI affordance in Settings. Batch transcription unaffected if streaming not downloaded.
EOU 120M accuracy vs TDT V3 batch accuracy Medium Must benchmark before shipping Phase C. If EOU WER is unacceptable, final result falls back to batch transcription of accumulated audio buffer.
Actor isolation deadlock — ParakeetTranscriber + StreamingEouAsrManager Medium Both are actors. Chunk forwarding must use Task { await streamingManager.appendAudio(...) } inside ParakeetTranscriber, not a synchronous bridge. Careful design in Phase B.
AVAudioEngine tap cadence vs EOU chunk size mismatch Low AudioRecorder tap fires every ~64ms (bufferSize 1024 at 16kHz). EOU expects 160ms chunks. Accumulate 2–3 tap callbacks before forwarding. Implement in the onAudioChunk accumulator.
Ghost text diverges from final transcript Low Final endStreaming() result is authoritative. Overlay updates to confirmed text at paste time. Acceptable UX tradeoff — clearly communicated by ghost text opacity.
WhisperKit not streaming-capable in this implementation Low WhisperKit falls back to existing batch path. supportsStreaming returns false. No regression.
